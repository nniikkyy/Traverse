# Submission Evidence Checklist

Use this file to finalize your university submission package on GitHub.

## 1) Demo Video (2-5 minutes)

Upload one of the following to your repo:

- `docs/evidence/demo-video-link.txt` (Google Drive / YouTube unlisted link)
- or `docs/evidence/demo.mp4` (if file size allows)

Suggested demo flow:

1. Open home page and explain project goal.
2. Show signup/dashboard/match/chat/itinerary flow.
3. Send one AI chat prompt and show returned host + itinerary.
4. Show backend endpoint call (`/ai/agent-chat`).
5. End with architecture summary.

## 2) Screenshots To Add

Create folder `docs/evidence/screenshots/` and add:

1. `01-homepage.png`
2. `02-dashboard.png`
3. `03-host-match.png`
4. `04-agent-chat-output.png`
5. `05-itinerary.png`
6. `06-api-response.png`

## 3) What Examiners Should Review

- Project overview and run steps: `README.md`
- Architecture: `docs/architecture.md`
- API samples: `docs/api-examples.md`
- Source code:
  - Frontend: `app/`, `components/`
  - Backend: `local-host-ai/backend/`

## 4) Final Submission Checklist

- [ ] GitHub repo link shared
- [ ] README complete and accurate
- [ ] Architecture diagram included
- [ ] API request/response examples included
- [ ] Demo video link/file added
- [ ] Screenshots added
- [ ] Private repo access granted to evaluator (if repo is private)
